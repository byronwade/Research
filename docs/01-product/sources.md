# Sources

Sources are a first-class Project surface and the foundation of every grounded answer, claim, document, artifact, and publication.

## Supported source families

- Files: PDF, Office formats, Markdown, text, images, email exports, archives, and structured data.
- Media: audio, video, captions, transcripts, and extracted frames.
- Web: URLs, crawl collections, search results, snapshots, and monitored pages.
- GitHub: repositories, commits, files, pull requests, issues, releases, and discussions.
- Connected services: Google Workspace, Microsoft 365, email, collaboration tools, databases, and MCP servers.
- Scholarly systems: Crossref, OpenAlex, Semantic Scholar, PubMed, Europe PMC, arXiv, Unpaywall, OpenCitations, ORCID, ROR, and DataCite.

## Source lifecycle

```text
Candidate → Active → Pinned / Stale / Quarantined / Excluded / Archived
```

Agents may propose source additions, replacements, exclusion, or archival. They cannot silently delete user uploads or override pinned sources.

## Source card

A source card exposes title, type, owner, visibility, connection, synchronization state, version, freshness, capabilities, claims supported, last use, and audit history.

## Explicit capabilities

Every source records whether it is:

- stored;
- inspectable;
- searchable;
- citable;
- and richly understood.

The interface never implies that all uploaded formats receive equivalent semantic understanding.

## Exact citations

Clicking a citation opens the exact source locator: PDF page and bounding box, web snapshot passage, spreadsheet range, media timecode, Git commit and line range, email message, or connected-record identifier.

## Permissions and privacy

The same internal source can support private and public document projections, but publication policy decides what may be exposed or quoted. Authorization is applied before retrieval, and source revocation invalidates inaccessible evidence in dependent outputs.
