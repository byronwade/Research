# Start here

This is the canonical navigation entry for humans and coding agents building Research.

## Product in one sentence

Research turns Project conversations and authorized sources into durable, continuously maintained Markdown documentation backed by exact evidence.

## Primary user model

```text
Project
├── Chat
├── Documents
└── Sources
```

Chat is the control plane. Documents are the durable product. Sources are the evidence system. Command Center, Focus, Project Atlas, Source-change Maintenance, Scenario Lab, Reversible Work, Native Companion, Studio, Evidence, Memory, Research Runs, Automation Registry, Automation Recipes, GitHub, Activity, Publish, and Settings are contextual views around those three primary surfaces.

## Agent workflow

```bash
node scripts/agent-status.mjs
node scripts/agent-context.mjs
pnpm docs:check
```

The first command identifies the next dependency-eligible implementation slice. The second prints the exact documents and requirements needed for that slice.

## Current implementation truth

The specification is mature; the runtime is not yet scaffolded. The first executable slice is `foundation-01`, governed by `FND-001` through `FND-003`: pinned pnpm/Turborepo workspace, TanStack Start application, Hono API boundary, strict TypeScript configuration, deterministic bootstrap, provider simulators, quality gates, supply-chain controls, provenance evidence, and CI.

The first product proof after foundation work is deliberately narrow:

```text
Create Project
→ upload one PDF
→ create an immutable source version
→ parse and index it
→ ask a question in Chat
→ retrieve exact evidence
→ stream a cited answer
→ create an editable Markdown document
→ persist and reopen it
```

Do not begin with broad connector coverage, an uncontrolled agent swarm, dozens of artifact types, billing, or enterprise administration before this grounded vertical path works.
