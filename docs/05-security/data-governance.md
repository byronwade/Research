# Data governance

## Data classification

Organizations and Projects classify source material, generated content, credentials, telemetry, native companion grants, context packets, companion queue metadata, accessibility preferences, locale preferences, adaptive preference records, preference observations, translations, accessible-output manifests, and exports. Classification determines permitted providers, regions, connectors, retention, publication, support access, companion use, and model use.

## Provider governance

Every model, search service, translation service, transcription service, OCR service, parser, research engine, connector intermediary, and execution service has a versioned provider profile containing retention, training use, supported data classes, regions, subprocessors, security posture, contractual status, and permitted Project classifications.

A provider is selected only when both capability policy and data-governance policy allow it.

## Content rights

Accessibility on the web does not automatically grant permission to ingest, quote, redistribute, or publish. A SourceVersion records acquisition basis, rights status, license or terms, attribution requirements, quotation limits, redistribution rules, review date, and takedown path.

## Residency

The system records the storage and processing region for raw sources, derived text, translations, transcriptions, OCR output, embeddings, model context, workflow state, backups, and exports. Cross-region processing requires explicit policy authorization.

## Support access

Support is metadata-first. SupportDiagnosticBundles are customer-visible, expiring, content-minimized, and revocable where policy allows. Any content access is least-privilege, time-limited, case-bound, approved through a SupportAccessRequest where required, represented as a SupportAccessSession, customer-visible, and audited. Support tools do not grant ambient tenant access, silent impersonation, arbitrary export, arbitrary SQL, hidden mutation, or private-content model evaluation.

## Analytics and feedback

Product analytics uses an allowlisted event schema governed by [`../06-delivery/telemetry-and-experience-instrumentation-matrix.md`](../06-delivery/telemetry-and-experience-instrumentation-matrix.md). Raw prompts, source bodies, document bodies, citations, credentials, private tool output, screen captures, clipboard contents, full Resume Digest bodies, full Workset or pane content, full Partial Result content, local draft bodies, offline packet contents, browser history, and operating-system window state are prohibited unless a separately governed diagnostic workflow obtains explicit authorization.
Device continuity analytics store metadata such as capability class, installation context, viewport class, storage class, queue count, blocked reason, conflict kind, sync result, eviction state, and reconnect latency. They do not store local draft bodies, source text, prompts, private document bodies, citations, credentials, connector payloads, private URLs, screenshots, clipboard contents, browser history, operating-system state, hidden reasoning, or raw user-agent strings unless a specific support diagnostic policy allows it.
Native companion analytics store metadata such as surface type, install version class, grant state, capture mode, blocked reason, queue count, sync result, deep-link result, shortcut conflict class, extension update state, and revocation reason. They do not store selected text, active-tab contents, local file contents, raw file paths where redaction is required, raw URLs when private, browser history, screenshots, clipboard contents, keystrokes, camera, microphone, OS window state, prompts, document bodies, credentials, connector payloads, hidden reasoning, or raw provider traces.
Adaptive preference analytics store metadata such as preference class, scope, adaptation mode, explanation opened, suggestion accepted or dismissed, wrong-inference correction, reset/export use, policy-managed state, invalidation reason, and model-context inclusion class. They do not store raw source text, prompts, private document bodies, citations, private comments, credentials, connector payloads, private URLs, screenshots, clipboard contents, browser history, operating-system state, hidden reasoning, support notes, or raw provider traces.
Accessibility and locale analytics store metadata such as validation failure class, route, surface, language tag, direction class, locale class, export format, manifest status, and unsupported-language limitation. They do not store raw source text, prompts, document bodies, citations, private comments, credentials, assistive-technology identity, disability status, private URLs, screenshots, clipboard contents, browser history, operating-system state, or hidden reasoning.
Approval-load and delegated-trust analytics store metadata such as action class, prompt count, grant state, decision kind, stale-receipt result, and outcome link. They do not store private source text, raw prompt bodies, document bodies, connector payloads, credentials, or hidden reasoning.
Maintenance analytics store metadata such as stale-claim resolution, patch acceptance, false-positive class, false-negative class, publication-blocker class, cost, latency, approval burden, and user correction. They do not store private source text, raw prompt bodies, private document bodies, connector payloads, credentials, unsupported private claim text, or hidden reasoning.
Project Health and support analytics store metadata such as health domain, reason category, severity, redaction class, causal lineage class, suspected-cause confidence, counterevidence class, unknown-state class, false-cause class, diagnostic-waste class, repair class, dry-run result, approval class, SupportDiagnosticBundle status, SupportAccessRequest decision, SupportAccessSession duration, revocation, break-glass class, latency, repeated-repair count, and outcome classification. They do not store private source text, raw prompt bodies, raw completion bodies, raw tool payloads, document bodies, connector payloads, credentials, screenshots, clipboard contents, browser history, operating-system state, raw OpenTelemetry span bodies, raw GenAI trace content, unredacted support notes, or hidden reasoning.
Scenario analytics store metadata such as scenario type, option count, simulated effect classes, unknown count, live-test class, stale-plan rejection, apply-candidate state, approval class, cost and latency class, user correction, and outcome classification. They do not store private source text, raw prompt bodies, document bodies, connector payloads, credentials, screenshots, clipboard contents, browser history, operating-system state, private URLs, unredacted support notes, or hidden reasoning.
Abuse analytics store metadata such as decision state, reason category, action class, route class, provider-policy result, throttle class, review state, appeal outcome, false-positive classification, emergency-control use, quota pressure, retry class, and support-safe outcome link. They do not store private source text, raw prompt bodies, document bodies, connector payloads, credentials, private URLs, screenshots, clipboard contents, browser history, operating-system state, unredacted support notes, hidden reasoning, or raw provider traces.
